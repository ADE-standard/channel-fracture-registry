# CF-0102: langgraph dev: checkpoints never flushed mid-session; data loss on non-graceful exit

| 字段 | 值 |
|------|-----|
| **CF-ID** | CF-0102 |
| **类别** | State loss |
| **影响系统** | LangGraph |
| **ADE层级** | L2 |
| **故障模式** | Checkpoint Flush Loop Silent Drop |
| **触发条件** | Non-graceful exit (SIGKILL/crash/power loss) of langgraph dev server |
| **观察症状** | All thread state gone after crash; empty PersistentDicts silently dropped from flush loop; 500 errors on state/history |
| **根因** | Flush loop classifies empty-but-alive PersistentDicts as dead weakrefs (falsy dict); unregisters checkpoint stores on first tick |
| **检测方法** | CADVP CC-4 checkpoint persistence verification on non-graceful exit |
| **影响** | Complete data loss since last graceful shutdown; all checkpoint durability guarantees broken |
| **缓解措施** | Fix falsy dict check in flush loop; always write checkpoints on tick regardless of emptiness |
| **证据** | [https://github.com/langchain-ai/langgraph/issues/8298](https://github.com/langchain-ai/langgraph/issues/8298) |
| **状态** | Open |
| **分级** | Tier B |
| **分类** | state |
| **证据等级** | reported |
