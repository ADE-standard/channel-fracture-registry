# CF-0066

## Title
Async tasks receive empty context due to last_sync_output truncation

## Category
Memory blindness

## Affected System
CrewAI

## Layer
L2

## Failure Mode
Context Truncation Across Async Boundary

## Trigger
Task with async_execution=True in multi-task crew

## Observed Symptom
Async task receives empty context; prior outputs invisible

## Root Cause
last_sync_output only populated on conditional-resume, not on normal async start

## Detection
CADVP CC-1 memory availability at async task startup

## Impact
False belief of cross-task knowledge flow; async tasks blind to prior results

## Mitigation
Populate async context from all completed preceding tasks

arxiv:2606.04896
