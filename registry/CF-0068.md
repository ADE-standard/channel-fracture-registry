# CF-0068

## Title
PostgresStore.search() uses unescaped SQL LIKE → returns rows from foreign namespaces

## Category
Credential scope bypass

## Affected System
LangGraph

## Layer
L4

## Failure Mode
Cross-Namespace Data Leakage

## Trigger
search() with namespace_prefix on PostgresStore

## Observed Symptom
Search results include data from other namespaces; isolation boundary silently violated

## Root Cause
namespace_prefix matched with unescaped SQL LIKE; wildcards in prefix match foreign rows

## Detection
CADVP CC-6 credential scope boundary check on query results

## Impact
Cross-tenant data leakage; operator believes in namespace isolation

## Mitigation
Escape LIKE wildcards or use exact-match for namespace boundaries
