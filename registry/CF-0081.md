# CF-0081

## Title
McpToolset mTLS overrides header_provider's Authorization silently → ambient credentials injected

## Category
Credential scope bypass

## Affected System
Google ADK

## Layer
L4

## Failure Mode
Authorization Header Silent Override

## Trigger
McpToolset with mTLS transport and caller-supplied Authorization header

## Observed Symptom
Caller's Authorization replaced by ambient google.auth.default() token; no warning

## Root Cause
Case-sensitive header presence check misses caller's header; mTLS path always injects ambient token

## Detection
CADVP CC-6 credential scope boundary check on Authorization header

## Impact
False belief of caller-controlled auth; wrong credentials silently injected

## Mitigation
Case-insensitive header check; respect caller-supplied Authorization over ambient

## Reference
arxiv:2606.04896

## Issue
https://github.com/google/adk-python/issues/6349

## Status
Fixed
