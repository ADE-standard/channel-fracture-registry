# CF-0059

## Title
Deleting thread leaks channel blobs → unbounded memory growth in long-running servers

## Category
Process lifecycle

## Affected System
LangGraph

## Layer
L5

## Failure Mode
Resource Leak via Deletion

## Trigger
Thread deletion during long-running server

## Observed Symptom
Memory grows monotonically; deleted threads still consume RAM

## Root Cause
Thread deletion removes registry entry but not channel blob references

## Detection
CADVP CC-8 state file integrity + memory profiling

## Impact
Silent unbounded memory leak; eventual OOM

## Mitigation
Release all channel blob references on thread deletion

arxiv:2606.04896
