# CF-0076

## Title
model_to_tools router returns 'model' but path_map omits it → KeyError silently aborts

## Category
Tool surface illusion

## Affected System
LangChain

## Layer
L3

## Failure Mode
Router-Path Map Mismatch

## Trigger
create_agent with model_to_tools routing

## Observed Symptom
Router selects 'model' path; path_map lookup fails KeyError; agent silently fails

## Root Cause
model_to_tools router can return 'model' but path_map does not include 'model' key

## Detection
CADVP CC-7 tool availability probe on routing table

## Impact
False belief of complete routing; agent silently aborts when 'model' route selected

## Mitigation
Add 'model' to path_map or prevent router from returning unmatched paths
