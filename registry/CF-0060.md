# CF-0060

## Title
put_writes/put persistence order unenforced → post-crash recovery host-dependent

## Category
Process lifecycle

## Affected System
LangGraph

## Layer
L4

## Failure Mode
Non-Deterministic Crash Recovery

## Trigger
Machine crash during execution with durability='sync'

## Observed Symptom
Recovery path varies by host timing; different supersteps replayed per machine

## Root Cause
No guaranteed ordering between writes and checkpoint persistence

## Detection
CADVP CC-8 transaction boundary verification

## Impact
False belief of deterministic recovery; state diverges across failover

## Mitigation
Wrap put_writes + checkpoint into atomic transaction

arxiv:2606.04896
