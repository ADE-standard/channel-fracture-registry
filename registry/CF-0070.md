# CF-0070

## Title
_resolve_schemas processes repeated state schema at FIRST position → silent field constraint loss

## Category
Config override

## Affected System
LangChain

## Layer
L2

## Failure Mode
Schema Field Erasure via Position

## Trigger
Repeated state schema processed by _resolve_schemas

## Observed Symptom
Explicit state_schema loses field constraints silently; only first-position schema respected

## Root Cause
_resolve_schemas resolves repeated schema at first occurrence, ignoring later explicit constraints

## Detection
CADVP CC-2 config vs runtime schema diff

## Impact
False belief of schema enforcement; undefined fields pass validation

## Mitigation
Merge repeated schemas or warn on constraint shadowing
