# CF-0095

## Title
RedisJsonCollection.delete() silently fails when prefix_collection_name_to_key_names enabled

## Category
Process lifecycle

## Affected System
Semantic Kernel

## Layer
L2

## Failure Mode
Silent Delete Failure via Key Prefix

## Trigger
delete() called with prefix_collection_name_to_key_names=True

## Observed Symptom
Delete returns success but key remains; upsert path prefixed keys but delete path doesn't

## Root Cause
_inner_delete sends JSON.DEL {key} without collection-name prefix; targets non-existent key silently

## Detection
CADVP CC-8 state file integrity verification after delete

## Impact
False belief of data deletion; stale keys persist in Redis without awareness

## Mitigation
Apply collection-name prefix consistently across upsert and delete paths
