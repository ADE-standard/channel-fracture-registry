# CF-0055

## Title
durability='sync': put_writes/put order unenforced → inconsistent crash recovery

## Category
Config override

## Affected System
LangGraph

## Layer
L4

## Failure Mode
Checkpoint Ordering Violation

## Trigger
Crash recovery with durability='sync'

## Observed Symptom
Writes from partial superstep restored against wrong checkpoint → inconsistent state

## Root Cause
No guaranteed ordering between put_writes() and checkpoint persistence

## Detection
CADVP CC-2 config vs runtime diff

## Impact
False belief of crash-safe durability; state corruption passes integrity checks

## Mitigation
Enforce write-before-checkpoint ordering with transaction barrier

arxiv:2606.04896
