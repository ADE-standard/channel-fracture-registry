# CF-0088

## Title
@listen(or_(A,B,C)) multi-source OR listener only fires once → blocking cyclic flow re-triggering

## Category
Hook not firing

## Affected System
CrewAI

## Layer
L3

## Failure Mode
Multi-Source Listener Fire-Once Bug

## Trigger
OR listener (@listen(or_(A,B,C))) configured for multiple task completion events

## Observed Symptom
Listener fires once then silently ignores subsequent trigger events; cyclic workflow stalls

## Root Cause
Multi-source OR listener lacks re-trigger logic; fires once and transitions to 'completed' state

## Detection
CADVP CC-3 hook execution counter on multi-source listener

## Impact
False belief of multi-event listener; cyclic/callback workflows silently stop after first trigger

## Mitigation
Reset listener state or support multi-fire mode for OR listeners
