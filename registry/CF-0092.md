# CF-0092

## Title
Task(human_input=True) raises AttributeError after default executor swap → silent human-in-the-loop break

## Category
Config override

## Affected System
CrewAI

## Layer
L3

## Failure Mode
Executor Swap Breaks Human Input

## Trigger
Task configured with human_input=True after CrewAgentExecutor→AgentExecutor swap in v1.15.0

## Observed Symptom
AttributeError crashes flow; human feedback mechanism silently broken by version upgrade

## Root Cause
AgentExecutor lacks ask_for_human_input method that CrewAgentExecutor had; swap was not backward-compatible

## Detection
CADVP CC-2 config vs runtime executor capability verification

## Impact
False belief of HITL support; upgrade silently disables human feedback without warning

## Mitigation
Migrate HITL method during executor swap or maintain backward compatibility
