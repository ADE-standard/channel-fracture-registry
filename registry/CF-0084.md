# CF-0084

## Title
ChatHistory broken when LLM calls multiple tools in one message → history unusable for subsequent requests

## Category
Silent message loss

## Affected System
Semantic Kernel

## Layer
L3

## Failure Mode
Multi-Tool History Corruption

## Trigger
LLM (Claude Sonnet) calls multiple tools in single message via Bedrock connector

## Observed Symptom
ChatHistory becomes unparseable; subsequent requests fail with toolResult ID mismatch

## Root Cause
Agent builds history with multi-tool invocation in a format not consumable by subsequent requests

## Detection
CADVP CC-4 end-to-end content hash on chat history

## Impact
False belief of persistent conversation; history silently corrupted after multi-tool turns

## Mitigation
Normalize multi-tool message format for replay compatibility
