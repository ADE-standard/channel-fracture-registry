# CF-0057

## Title
Checkpoint serialization downcasts defaultdict/Counter to plain dict → losing default_factory

## Category
Tool surface illusion

## Affected System
LangGraph

## Layer
L3

## Failure Mode
Type Erasure via Serialization

## Trigger
State with defaultdict/Counter is checkpointed and restored

## Observed Symptom
After restore, KeyError instead of default; behavioral change

## Root Cause
Serializer treats dict subclasses as plain dict, discarding subclass behavior

## Detection
CADVP CC-4 content hash with type fidelity

## Impact
False belief of type-safe serialization; silent behavioral change

## Mitigation
Preserve dict subclass identity during serialization
