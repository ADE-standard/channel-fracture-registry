# CF-0090

## Title
Reasoning plan always detects 'NOT READY' even when model indicates 'READY' → plan stuck in loop

## Category
Hook not firing

## Affected System
CrewAI

## Layer
L3

## Failure Mode
Reasoning Plan False Negative

## Trigger
Model returns 'READY' status in reasoning plan output

## Observed Symptom
System classifies output as 'NOT READY'; agent loops indefinitely on already-complete plan

## Root Cause
READY detection logic fails to match model's actual READY signal format; mismatch silently loops

## Detection
CADVP CC-3 hook execution counter on reasoning completion path

## Impact
False belief of incomplete reasoning; agent stuck in infinite planning loop

## Mitigation
Normalize READY detection to accept multiple output formats
