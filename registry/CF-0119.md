# CF-0119: Streaming tool-call JSON truncated mid-stream — write_file silently replaced with empty object

## Source
- **Model:** Doubao seed-2-1-pro-260628 / seed-2-1-turbo-260628
- **Consumer:** Hermes Agent (chat_completion_helpers.py L2618: hardcoded `stream=True`)
- **Status:** Confirmed
- **Tier:** A (production-impacting, user-facing silence)
- **Framework:** Hermes Agent + Volcengine Doubao API

## CFR Classification
- **Failure Mode:** C3 假信信号 — model emits tool_call delta chunks, JSON truncated mid-stream; Hermes parser substitutes empty object
- **Taxonomy:** tool

## Diagnostic Signatures
1. Agent log: `WARNING agent.message_sanitization: Unrepairable tool_call arguments for write_file — replaced with empty object`
2. Cumulative count: `grep -a 'Unrepairable tool_call' agent.log | wc -l` → 30+ (single session)
3. User perception: agent "没反应" (unresponsive) — file never written, no error surfaced to user
4. Model enters retry loop: same write_file attempted repeatedly, same truncation each time

## Reproduction
```bash
# Fails on seed-2-1 with stream=true:
curl -s https://ark.cn-beijing.volces.com/api/v3/chat/completions \
  -H "Authorization: Bearer $ARK_API_KEY" \
  -d '{"model":"doubao-seed-2-1-turbo-260628","stream":true,
       "messages":[{"role":"user","content":"write_file tool_call with 500-char HTML"}],
       "tools":[{"type":"function","function":{"name":"write_file","parameters":{...}}}],
       "max_tokens":8192}'
# → JSON truncated mid-stream

# Passes on seed-2-0 with same parameters:
# → JSON complete and parseable

# Passes on seed-2-1 with stream=false:
# → JSON complete and parseable
```

## Boundary Analysis
- **Model side (Volcengine):** Seed-2-1 streaming output truncates nested JSON tool_call arguments when content length exceeds ~1000 chars. Seed-2-0 does not exhibit this.
- **Consumer side (Hermes):** `chat_completion_helpers.py` L2618 hardcodes `stream=True` for all providers. No configuration path exists to request non-streaming. When JSON is truncated, Hermes substitutes an empty object and silently continues — the tool invocation fails but the agent does not know why.

## Cross-Framework Relevance
This fracture pattern applies to any agent framework that:
1. Uses streaming API for tool-calling LLMs
2. Has no non-streaming fallback path
3. Silently substitutes malformed tool_call parameters instead of surfacing the parse error

Potential candidates: LangGraph, AutoGen, CrewAI (all use streaming tool-calling).

## Mitigation
- **Short-term:** Use doubao-seed-2-0-pro-260215 (confirmed stable streaming JSON output)
- **Long-term:** Hermes should support `stream: false` provider-level config for providers known to have streaming tool_call issues, or auto-detect truncation and retry non-streaming
