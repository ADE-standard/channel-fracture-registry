# CF-0104: Run Cancellation Causes Loss of Streamed State Not Yet Persisted as a Checkpoint

| 字段 | 值 |
|------|-----|
| **CF-ID** | CF-0104 |
| **类别** | State loss |
| **影响系统** | LangGraph |
| **ADE层级** | L2 |
| **故障模式** | Cancel-Induced State Loss |
| **触发条件** | Canceling a streaming LangGraph run before checkpoint is persisted |
| **观察症状** | Streamed state between last persisted checkpoint and cancel is silently lost; UI shows inconsistent state |
| **根因** | No persistence on cancellation path; streamed intermediate states are never checkpointed during abort |
| **检测方法** | CADVP CC-1 checkpoint write count on cancellation |
| **影响** | Loss of unpersisted state; UI desynchronization; users must manually reconstruct lost state |
| **缓解措施** | Add checkpoint flush hook on cancellation; persist in-progress state before abort |
| **证据** | [https://github.com/langchain-ai/langgraph/issues/5672](https://github.com/langchain-ai/langgraph/issues/5672) |
| **状态** | Open |
| **分级** | Tier B |
| **分类** | state |
| **证据等级** | reported |
