# CF-0109: Google ADK: Failed A2A task errors silently leak into conversation history as regular content

| 字段 | 值 |
|------|-----|
| **CF-ID** | CF-0109 |
| **类别** | Error boundary leak |
| **影响系统** | Google ADK |
| **ADE层级** | L3 |
| **故障模式** | Error Content Boundary Violation |
| **触发条件** | Remote A2A task returns state:'failed'; RemoteA2aAgent creates Event with content instead of error_message |
| **观察症状** | Failed task errors appear as regular conversation content; agent treats error text as valid input |
| **根因** | RemoteA2aAgent creates Event with error details in content field instead of error_message; no error/content separation |
| **检测方法** | CADVP CC-6 error boundary validation on A2A task failures |
| **影响** | Agents reason about error text as factual content; cascading hallucination from error leakage |
| **缓解措施** | Create Event with error_message set instead of content when task state is 'failed' |
| **证据** | [https://github.com/google/adk-python/issues/4309](https://github.com/google/adk-python/issues/4309) |
| **状态** | Fixed |
| **分级** | Tier B |
| **分类** | communication |
| **证据等级** | reported |
